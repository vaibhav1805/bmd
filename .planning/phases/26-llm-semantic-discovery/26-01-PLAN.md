**Fix 1 — Pointer bug at line 190 (required, definitive):**
The current code reads:
```go
cache.Entries = append(cache.Entries, entry)
cacheByDocID[entry.DocID] = &entry
```
`&entry` takes the address of the range variable, which is overwritten on the next loop iteration. Replace these two lines with:
```go
cache.Entries = append(cache.Entries, entry)
cacheByDocID[entry.DocID] = &cache.Entries[len(cache.Entries)-1]
```
This takes the address of the newly-appended element in the slice, which is stable.
