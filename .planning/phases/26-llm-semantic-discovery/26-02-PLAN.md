  <action>
The current uncommitted diff in commands.go has a critical problem: `checkPageIndexAvailable` is called unconditionally at the start of `CmdIndex`, which breaks users without pageindex. Fix commands.go to make LLM discovery opt-in:

**Changes to make in CmdIndex:**

1. **Add --llm-discovery flag to IndexArgs struct** (if not already present):
   ```go
   // LLMDiscovery enables LLM-powered semantic discovery via PageIndex.
   // When false (default), LLMSemanticRelationships is skipped.
   LLMDiscovery bool
   ```

2. **Wire the flag in the argument parser** (wherever the other index flags are registered, e.g., using flag.BoolVar):
   ```go
   fs.BoolVar(&a.LLMDiscovery, "llm-discovery", false, "enable LLM-powered semantic discovery via PageIndex (requires pageindex CLI)")
   ```

3. **In CmdIndex, move the PageIndex availability check behind the flag**:
   Remove the unconditional `checkPageIndexAvailable` call near the top. Replace the entire block that unconditionally checks pageindex with:
   ```go
   llmCfg := DefaultLLMDiscoveryConfig()
   if a.PageIndexBin != "" {
       llmCfg.ExecutablePath = a.PageIndexBin
   }
   if a.Model != "" {
       llmCfg.Model = a.Model
   }
   if a.LLMDiscovery {
       if err := checkPageIndexAvailable(llmCfg.ExecutablePath); err != nil {
           return fmt.Errorf("index --llm-discovery: %w", err)
       }
   }
   ```

4. **Conditionalize PageIndex tree generation** — the block that calls RunPageIndex in a loop must be guarded:
   ```go
   var trees []FileTree
   var knownComponents []string
   if a.LLMDiscovery && !a.SkipDiscovery {
       fmt.Fprintf(os.Stderr, "  Generating PageIndex trees for LLM discovery...\n")
       // ... existing RunPageIndex loop ...
       // ... existing knownComponents extraction ...
   }
   ```
   When `!a.LLMDiscovery`, trees and knownComponents remain empty slices/nil — LLMSemanticRelationships will return nil for empty inputs (as verified in Plan 01 tests), so the 4th algorithm simply contributes no edges.

5. **Remove the duplicate service detection** that was added for knownComponents: after LLM discovery, `sd.DetectComponents(graph, docs)` is called again. This double-detection is wasteful. Keep only the post-discovery call (which uses the updated graph). Move knownComponents extraction to only happen inside the `if a.LLMDiscovery` block before discovery runs.

6. **Remove the dead conditional** at the bottom that was removed in the diff:
   The block `if strings.ToLower(a.Strategy) == "pageindex"` for PageIndex tree generation has been removed in the diff — confirm it is gone from commands.go. Tree generation is now handled in the LLM discovery block above.

7. **Ensure the `errors` import is still used** after removing the old conditional block. If the only use of `errors` was `errors.Is(err, ErrPageIndexNotFound)` in the removed block, add a blank import or remove the import. Run `go build` to verify.

After applying changes, run `go build ./...` to confirm compilation.
