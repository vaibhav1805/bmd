Create internal/knowledge/context.go (package knowledge):

1. ContextSection struct — one retrieved section:
   ```go
   type ContextSection struct {
       File        string // relative file path, e.g. "auth-service.md"
       HeadingPath string // e.g. "Database Layer" or "" for preamble
       Content     string // raw chunk content (markdown)
       Score       float64 // relevance score from retrieval
   }
   ```

2. AssembleContextBlock(query string, sections []ContextSection) string
   Produces the markdown format specified in CONTEXT.md:
   ```
   # Context for: {query}

   ## [{file} § {heading}]
   {content}

   ## [{file} § {heading}]
   {content}
   ```
   Rules:
   - Header line: "# Context for: " + query
   - For each section, citation: "## [" + section.File + " § " + section.HeadingPath + "]"
   - If HeadingPath == "", use "## [" + section.File + "]" (no § separator for preamble)
   - Append section.Content verbatim (no trimming)
   - Sections separated by blank line
   - Returns empty string "" when sections is nil/empty (caller handles no-results case)

3. sectionsFromBM25Results(results []SearchResult) []ContextSection
   Converts BM25 SearchResult slice to []ContextSection for use in fallback path.
   Maps: RelPath → File, HeadingPath → HeadingPath, ContentPreview → Content, Score → Score

Add to internal/knowledge/context.go:

1. ContextArgs struct:
   ```go
   type ContextArgs struct {
       Query  string
       Dir    string
       Top    int    // max sections to return, default 5
       Format string // "markdown" (default) | "json"
       Model  string // LLM model, default "claude-sonnet-4-5"
   }
   ```

2. ParseContextArgs(args []string) (*ContextArgs, error)
   - Uses splitPositionalsAndFlags (same pattern as ParseQueryArgs)
   - Flags: --dir (default "."), --top (default 5), --format (default "markdown"), --model (default "claude-sonnet-4-5")
   - Positional: first arg is the query string
   - Returns error if no query provided

3. CmdContext(args []string) error
   Implementation:
   a. Parse args via ParseContextArgs
   b. Resolve absDir via filepath.Abs
   c. Attempt to LoadTreeFiles(absDir) — if trees loaded (len > 0):
      - Log to stderr: "Using PageIndex trees (%d files)\n"
      - Invoke pageindex query subprocess:
        RunPageIndexQuery(PageIndexConfig{ExecutablePath: "pageindex", Model: a.Model}, a.Query, trees, a.Top)
        This returns []ContextSection; see step 4 below
   d. Fallback (no trees or RunPageIndexQuery error):
      - Log to stderr: "No .bmd-tree.json files found; falling back to BM25 chunk search\n"
      - openOrBuildIndex → LoadIndex → idx.Search(a.Query, a.Top)
      - Convert via sectionsFromBM25Results
   e. If no sections: print "No relevant context found for: {query}\n"; return nil
   f. Format and print:
      - format=="json": marshal ContractResponse with data={"query": q, "sections": [...], "count": N}
        Each section JSON: {"file": ..., "heading_path": ..., "content": ..., "score": ...}
      - format=="markdown" (default): print AssembleContextBlock(a.Query, sections)

4. Add RunPageIndexQuery to pageindex.go (since it depends on pageindex.go's config):
   ```go
   // RunPageIndexQuery invokes pageindex to search an existing tree index for the query.
   // It calls: pageindex query --query QUERY --model MODEL --top N --format json
   // stdin is fed the tree JSON list (as a JSON array).
   // Returns []ContextSection ordered by relevance score descending.
   func RunPageIndexQuery(cfg PageIndexConfig, query string, trees []FileTree, top int) ([]ContextSection, error)
   ```
   Implementation:
   - Marshal trees to JSON, pass via stdin to subprocess
   - Call: pageindex query --query QUERY --model MODEL --top TOP --format json
   - Parse stdout as JSON array of {file, heading_path, content, score}
   - Return []ContextSection
   - If executable not found: wrap with ErrPageIndexNotFound
   - If subprocess fails: return nil, err (caller falls back to BM25)
