    Create internal/mcp/watch_state.go in package mcp.

    Purpose: manage the lifecycle of active watch sessions so multiple MCP clients
    (agents) can each have their own isolated watch session.

    Types:

    ```go
    // WatchNotification is a single change event delivered to an agent on poll.
    type WatchNotification struct {
        Kind    string `json:"kind"`     // "created" | "modified" | "deleted"
        Path    string `json:"path"`     // absolute path
        RelPath string `json:"rel_path"` // directory-relative path
    }

    // WatchSession tracks an active filesystem watch for one MCP client.
    type WatchSession struct {
        ID        string    `json:"session_id"`
        Dir       string    `json:"dir"`
        StartedAt time.Time `json:"started_at"`

        updater  *knowledge.IncrementalUpdater
        mu       sync.Mutex
        pending  []WatchNotification  // buffered notifications since last poll
    }

    // WatchSessionManager is a concurrent-safe store of active WatchSessions.
    type WatchSessionManager struct {
        mu       sync.RWMutex
        sessions map[string]*WatchSession
    }
    ```

    NewWatchSessionManager() *WatchSessionManager

    WatchSessionManager methods:
    - Create(dir string, updater *knowledge.IncrementalUpdater) *WatchSession
      - generates session ID: fmt.Sprintf("ws-%d", time.Now().UnixNano())
      - registers onChange callback on updater: updater.SetOnChange(func(evt) { session.append(evt) })
      - stores session in map, returns it
    - Get(id string) (*WatchSession, bool)
    - Delete(id string) — calls session.updater.Stop(), removes from map

    WatchSession methods:
    - append(evt knowledge.WatchEvent) — mutex-protected append to pending
    - DrainPending() []WatchNotification — mutex-protected drain (swap with empty slice), return drained

    WatchEventKind to string mapping:
    - knowledge.WatchCreated  → "created"
    - knowledge.WatchModified → "modified"
    - knowledge.WatchDeleted  → "deleted"

    Import knowledge package: "github.com/bmd/bmd/internal/knowledge"

    PART B — handlers.go additions:

    handleWatchStart(ctx, req):
    1. dir = ParseString(req, "dir", s.baseDir)
    2. intervalMs = ParseInt(req, "interval_ms", 500)
    3. absDir, err := absPath(dir) — return error response if fails
    4. Load existing index+graph from DB or build if absent (reuse loadGraph helper)
    5. Load registry: knowledge.LoadRegistry(filepath.Join(absDir, knowledge.RegistryFileName))
       If nil: create empty NewComponentRegistry()
    6. Create FileWatcher: knowledge.NewFileWatcher(absDir, time.Duration(intervalMs)*time.Millisecond)
    7. Create updater: knowledge.NewIncrementalUpdater(absDir, watcher, index, graph, registry, db, nil)
    8. session := s.watchMgr.Create(absDir, updater)
    9. watcher.Start(); updater.Start()
    10. Return JSON: {"session_id": session.ID, "dir": absDir, "interval_ms": intervalMs, "status": "watching"}

    handleWatchPoll(ctx, req):
    1. sessionID = ParseString(req, "session_id", "") — return error if empty
    2. session, ok := s.watchMgr.Get(sessionID) — return error if not found
    3. notifications := session.DrainPending()
    4. Return JSON: {"session_id": sessionID, "notifications": notifications, "count": len(notifications)}

    handleWatchStop(ctx, req):
    1. sessionID = ParseString(req, "session_id", "") — return error if empty
    2. _, ok := s.watchMgr.Get(sessionID) — return error if not found
    3. s.watchMgr.Delete(sessionID) — stops updater+watcher, removes session
    4. Return JSON: {"session_id": sessionID, "status": "stopped"}

    All responses wrapped in knowledge.NewOKResponse / NewErrorResponse (CONTRACT-01 compliant).
