<feature>
  <name>FileWatcher — polling-based .md change detection</name>
  <files>
    internal/knowledge/watcher.go
    internal/knowledge/watcher_test.go
  </files>
  <behavior>
    WatchEventKind: Created | Modified | Deleted (iota)

    FileWatcher struct fields:
      - dir string           — watched directory (absolute)
      - interval time.Duration  — poll interval (default 500ms)
      - Events chan WatchEvent   — buffered (32) outbound channel
      - stop chan struct{}       — internal stop signal
      - snapshot map[string]snapshotEntry — path → {mtime, size}

    WatchEvent struct:
      - Kind WatchEventKind
      - Path string          — absolute file path
      - RelPath string        — relative to watched dir (forward slashes)

    NewFileWatcher(dir string, interval time.Duration) *FileWatcher

    Start() — launches goroutine; calls poll() on every tick
    Stop()  — closes stop channel; goroutine exits; closes Events channel

    poll() algorithm:
      1. WalkDir(dir), skip hidden dirs (same list as hiddenDirs in scanner.go), only collect .md files
      2. Compare current snapshot to previous:
         - path in current but not snapshot → Created event
         - path in snapshot but not current → Deleted event
         - path in both, mtime or size changed → Modified event
      3. Update snapshot to current state
      4. Send events to Events channel (non-blocking: drop if channel full)

    Edge cases:
      - Non-existent dir: Start() returns without launching goroutine, Events channel closed
      - Rapid file churn: non-blocking send prevents goroutine stall
      - Stop() is idempotent: calling twice does not panic

    Test cases (RED first, then GREEN):
      - TestWatcher_Created: write new .md file, receive Created event within 2s
      - TestWatcher_Modified: write existing .md, modify it, receive Modified event within 2s
      - TestWatcher_Deleted: write .md, delete it, receive Deleted event within 2s
      - TestWatcher_IgnoresHidden: write .md inside .git subdir, no event emitted
      - TestWatcher_IgnoresNonMd: write .txt file, no event emitted
      - TestWatcher_StopClean: Stop() returns, Events channel closed, no goroutine leak (use goleak or timeout)
      - TestWatcher_Idempotent: Stop() called twice, no panic
  </behavior>
  <implementation>
    Use os.Stat + filepath.WalkDir for polling — no external dependencies.
    Use time.NewTicker for the poll interval.
    The goroutine pattern:
      ticker := time.NewTicker(w.interval)
      defer ticker.Stop()
      for {
        select {
        case &lt;-w.stop: close(w.Events); return
        case &lt;-ticker.C: w.poll()
        }
      }
    Import scanner.go's hiddenDirs map directly (same package) for consistent skip behavior.
    snapshotEntry: { ModTime time.Time; Size int64 }
    Build initial snapshot on first poll (before emitting events) so the watcher
    does not fire spurious Created events for files that pre-existed Start().
  </implementation>
</feature>
