## Decisions Made

- Used `claudeClient` interface instead of direct SDK calls — enables mock injection in tests without any real API calls or environment setup
- `TreeNodeFromMarkdown` builds tree with a stack-based algorithm in O(n) — simpler than recursive descent, handles all H1-H6 nesting correctly
- `extractFirstSentence` provides lightweight summary fallback without an LLM call during tree construction — callers needing LLM summaries call `GenerateSectionSummary` explicitly
- `DiscoverWithClaude` uses the same cache format as `LLMSemanticRelationships` for backward compatibility
- Kept `LLMSemanticRelationships` function in `llm_semantic.go` (not deleted) to avoid breaking any external callers, but it's no longer called from the main pipeline
